# Date: 26/03/2026

**Objective:** (Address Resolution Protocol) ARP  observation, ARP spoofing. 

**Observation:**
- ARP requests: "Who has [IP]?"
- ARP replies: "[IP] is at [MAC]"
- ARP table caches these mappings

**ARP Spoofing (MITM):**
- ARP Table Shows IP → MAC mappings for devices on your local network
  - `dynamic` = learned automatically
  - `static` = manually set (more secure)
- Attacker replies to ARP requests with their own MAC
- Traffic is intercepted, read, then forwarded
- No authentication in ARP, attack is hard to detect without monitoring
- Normal: One MAC per IP in ARP table
- Attack: Same IP appears with different MACs (especially gateway)
- Detection: Monitor for duplicate MAC claims or unexpected MAC changes

**Why This Matters:**
- On shared networks (coffee shop, office), ARP spoofing can steal data
- Detection requires monitoring for duplicate MACs or unusual ARP activity
