## Automating SSH Brute Force Detection with Bash & Cron

### Overview

A bash script to detect SSH brute force attacks by counting failed login attempts from each IP in the last hour. 
I automated it using **cron** (Linux task scheduler) to run every hour and log alerts automatically.

This mimics real SOC detection systems — no manual log checking, just automated alerts.
---
### 1. The Detection Script

**File:** `/home/maxubuntu/Desktop/scripts/check_automatic.sh`

```bash
#!/bin/bash
# Detects SSH brute force attempts from the last hour

LOG_FILE="/var/log/brute_alerts.log"
IP_LIMIT=3

sudo journalctl -u ssh --since "1 hour ago" | grep "Failed password" | awk '{print $11}' | sort | uniq -c | while read count ip; do
    if [ "$count" -gt "$IP_LIMIT" ]; then
        echo "$(date): ALERT - $ip had $count failed SSH attempts" | tee -a "$LOG_FILE"
    fi
done
```
**What it does**:

- Checks SSH logs from the last hour
- Extracts all IPs with failed password attempts
- Counts failures per IP
- Logs an alert for any IP with >3 failures

### 4. Automating with Cron
To run the script every hour, I added a cron job:
```bash
crontab -e
```
At the bottom of the file:
(Syntax: Minute - Hour - DayMonth - Month - DayWeek - Command)
```0 * * * * /home/maxubuntu/Desktop/scripts/check_automatic.sh```
**Result**: Script runs automatically every hour: at 1:00, 2:00, 3:00, etc.
To verify the cron status, I use: 
``crontab -l``
```bash
0 * * * * /home/maxubuntu/Desktop/scripts/check_automatic.sh
```

### 6. What I learned:
- Scripting: 	Shebang (#!/bin/bash), variables, if, while read, tee
- Log: analysis	journalctl -u ssh filters SSH logs; awk '{print $11}' extracts IPs
- Cron:	Schedules tasks; paths must be absolute; smallest unit is 1 minute
- sudo in cron: Can fail (no TTY); better to run cron as root or adjust permissions
- Alerting: Logging to file is simple; email or SIEM integration in the future
