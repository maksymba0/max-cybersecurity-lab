# Date: 31/03/2026

**Objective:** Reconnaissance (information gathering) — DNS enumeration and port scanning of Target VM, Wireshark Linux

**DNS Findings:**
- dota2.com resolves to: 2.16.206.139, 2.16.206.140
- valvesoftware.com name servers (authoritative DNS):
  - a24-64.akam.net
  - a2-64.akam.net
  - a7-66.akam.net
  - a9-66.akam.net
  - a1-194.akam.net
  - a22-67.akam.net
- Interesting observation: dota2.com is hosted on Akamai CDN (2.16.206.139 and .140). Port scan of these IPs shows ports 80 (HTTP) and 443 (HTTPS) open, expected for a web service.

**Ping Sweep Results:**
- Live hosts in 192.168.100.0/24:
  - 192.168.100.10 (Target VM)
  - 192.168.100.20 (Attacker VM)

**Port Scan Results (Target VM 192.168.100.10):**
- Open ports: 80 (HTTP)
- Service: nginx 1.24.0 (Ubuntu)
- OS guess: Linux (consistent with Ubuntu)
- 
**Wireshark Observation:**
- Captured port scan from Attacker to Target using filter `tcp.flags.syn == 1`
- Observed multiple SYN packets to different ports - the signature of a network scan
- Target responded with SYN-ACK on open port (80) and RST on closed ports
- Seeing attacker's request in nginx logs, syslogs do not show attacker's requests as expected.
**What I Learned:**
- Reconnaissance (gathering information about a target) is the first phase of any attack or security assessment
- DNS reveals infrastructure without ever touching the target
- Nmap provides a fingerprint of a system: its open ports, services, and OS
- Ping sweeps help map live hosts on a network
